# 営業ヒアリングロープレAI — システム概要

## 何を作っているか

**営業担当者向けのヒアリングロープレ練習サービス。**  
AIが実在しそうな企業担当者を演じ、ユーザー（営業担当）が課題を引き出す「ディスカバリー商談」の練習を行える。

業界ごとに異なるペルソナを用意し、最初は表面的な悩みしか話さない担当者が、適切な質問を重ねることで「真の課題」を明かしていく構造になっている。

## ペルソナ（業界別）

| 業界 | 担当者 | 会社 | 最終的に明かされる真課題 |
|------|--------|------|------------------------|
| 製造 | 田中 幸生（代表取締役社長） | 田中金属加工株式会社 | 社長自身のIT苦手意識・社内にDX推進人材がいない |
| 金融 | 中村 義雄（営業推進部長） | 中央地域信用金庫 | ベテラン行員の変化への抵抗・システム障害への恐怖感 |
| 流通 | 鈴木 雄一郎（取締役副社長） | 関東中央物流株式会社 | ドライバーのITへの拒否反応・過去の導入失敗トラウマ |
| 小売 | 山田 恵子（代表取締役） | 株式会社やまだストア | 薄利による投資余力の不足・社内の「またシステム？」疲弊感 |

## システムの動き

### 1. 業界選択画面
アプリを開くと4業界のカードが表示され、練習したい業界を選んでから商談画面に入る。

### 2. 商談フェーズシステム（バックエンドで管理）

```
挨拶 → 表面課題 → 中間課題 → 真課題
```

バックエンドが会話のターン数とユーザーの発言に含まれる「掘り下げキーワード」を計測し、フェーズを自動的に進める。

| フェーズ | 移行条件 | AIの振る舞い |
|---------|---------|------------|
| 挨拶 | 2ターン経過 | 世間話程度、課題に触れない |
| 表面課題 | 掘り下げ発言 2回以上 | 漠然とした非効率の悩みを話す |
| 中間課題 | 掘り下げ発言 4回以上（累計） | 具体的な業務上の問題点を話す |
| 真課題 | 掘り下げ発言 6回以上（累計） | 人・感情・組織の根本課題を明かす |

**掘り下げキーワードの例：**「なぜ」「具体的に」「どのよう」「背景」「原因」など

### 3. AI応答生成

フェーズに応じたシステムプロンプトと会話履歴をOpenAI APIに送信し、ペルソナとして自然な返答を生成する。  
「どこまで話してよいか」をプロンプトで制御することで、フェーズ外の情報は自然に話さないよう設計されている。

### 4. 音声機能

- **音声入力（STT）**：スペースキー長押し or マイクボタンで録音。Web Speech APIによりリアルタイムで文字に変換される。
- **音声出力（TTS）**：AIの返答が自動で読み上げられ、擬似的な会話感を演出する。ミュートボタンで切り替え可能。

## データフロー

```
[ユーザー発言]
    ↓
フロントエンド（React）
    ↓ POST /api/chat  { message, history, industry }
バックエンド（FastAPI）
    ↓ フェーズ判定（キーワード数・ターン数）
OpenAI API（gpt-4o-mini）
    ↓ ペルソナとして返答生成
バックエンド
    ↓ { message, phase }
フロントエンド
    ↓
チャット表示 + 音声読み上げ
```

---

## セッション管理

- バックエンドはセッションIDをキーにフェーズ状態をメモリで管理する
- フロントエンドはlocalStorageに会話履歴・業界・セッションIDを保存し、リロード後も継続できる
- 「業界選択に戻る」でセッションと履歴をリセットする
