# README責務分割の更新（2026-06-18）

## 目的

`要件定義.md`の内容を、実装時に参照しやすいように`frontend/README.md`と`backend/README.md`へ責務別に再配置した。

## 実施内容

- `frontend/README.md`
  - フロントエンドの責務（画面、PTT、表示制御、API連携）を明確化
  - バックエンド依存の契約（セッション終了後の更新内容、真の課題の表示条件）を追記
  - フロント開発者が見るべき判断基準を追記
- `backend/README.md`
  - バックエンドの責務（ドメイン管理、音声処理、終了時処理、外部連携）を明確化
  - フロントとの共有契約（APIでの表示担保、非表示制御、異常終了時の扱い）を追記
  - インフラ前提と変更時の確認観点を追記

## 責務切り分け方針

- フロントは「入力・表示・画面遷移・UX」に集中し、業務判定ロジックは持たない
- バックエンドは「状態遷移・判定・保存・連携」に集中し、正データを一元管理する
- 共有事項は両READMEに記載し、片方だけ読んでも連携前提を見落とさない構成にした
